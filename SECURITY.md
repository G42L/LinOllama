# Security Policy

## Supported Versions

Security fixes are generally provided for the latest stable release of LinOllama.

Older releases may not receive security updates.

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub Issues.

Instead:

* Open a GitHub Security Advisory (preferred), if enabled.
* Otherwise, contact the project maintainer privately through GitHub.

Please include:

* A description of the vulnerability.
* Steps to reproduce.
* Potential impact.
* Suggested mitigation (if known).
* Proof of concept, if applicable.

## What to Expect

After receiving a report, we will:

1. Acknowledge receipt as soon as practical.
2. Investigate the issue.
3. Determine its severity and impact.
4. Develop and test a fix.
5. Release a patched version.
6. Publicly disclose the issue once users have had a reasonable opportunity to update.

## Responsible Disclosure

We ask that researchers avoid publicly disclosing vulnerabilities until a fix has been released.

This helps protect users of LinOllama.

## Security Best Practices

Users are encouraged to:

* Download LinOllama only from the official GitHub repository.
* Keep LinOllama up to date.
* Verify releases when checksums or signatures are provided.
* Avoid running untrusted plugins or modified binaries.

Thank you for helping keep LinOllama secure.
