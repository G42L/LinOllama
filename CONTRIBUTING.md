# Contributing to LinOllama

First of all, thank you for considering contributing to **LinOllama**!

Whether you're fixing a bug, improving the documentation, designing the UI, or implementing a new feature, your contributions are greatly appreciated.

## Code of Conduct

By participating in this project, you agree to follow our Code of Conduct. Please read `CODE_OF_CONDUCT.md` before contributing.

## Before You Start

If you plan to work on a significant feature or architectural change, please open an Issue first to discuss your proposal. This helps avoid duplicated work and ensures the feature aligns with the project's direction.

## Reporting Bugs

When reporting a bug, please include:

* LinOllama version
* Linux distribution and version
* Desktop environment (GNOME, KDE, XFCE, etc.)
* Installation method
* Steps to reproduce
* Expected behavior
* Actual behavior
* Screenshots or logs if available

## Suggesting Features

Feature requests are welcome.

Please explain:

* The problem you're trying to solve
* Your proposed solution
* Possible alternatives
* Any relevant screenshots or mockups

## Development Guidelines

### Coding Style

* Write clean and readable code.
* Keep functions focused and reasonably small.
* Prefer modern C++.
* Follow existing naming conventions.
* Avoid unnecessary dependencies.
* Comment *why*, not *what*.

### Commits

Use descriptive commit messages.

Examples:

* Fix crash when loading model list
* Add keyboard shortcut for new chat
* Improve markdown rendering

### Pull Requests

Before submitting a Pull Request, please ensure:

* The project builds successfully.
* Existing functionality is not broken.
* New code follows the existing style.
* Documentation is updated if necessary.
* The Pull Request clearly explains the purpose of the changes.

Smaller Pull Requests are generally easier to review.

If you're unsure about anything, feel free to open a Discussion or Issue before starting work.
